Launch a new agent to handle complex, multistep tasks autonomously. You must specify a subagent_type parameter to select which agent type to use.

When to use: execute custom slash commands, or delegate work to specialized agents.
When NOT to use: reading a specific file (use Read/Glob), searching a class definition (use Glob), searching within 2-3 files (use Read).

Usage notes:
1. Launch multiple agents concurrently to maximize performance; use a single message with multiple tool uses
2. Agent output is not visible to the user — summarize results in a text message. Output includes task_id to reuse the same session.
3. Each invocation starts fresh unless you provide task_id. Fresh prompts must be highly detailed with exactly what to return.
4. Tell the agent whether to write code or just do research, and how to verify its work.
5. Use agents proactively when their description matches the task.
